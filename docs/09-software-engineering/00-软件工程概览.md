# 软件工程核心知识

> **学习目标**：从"写代码的人"升级到"做工程的人"——解决的不是"代码能不能跑"，而是"系统能不能长期演进、团队能不能高效协作"
>
> **检验标准**：学完每个模块后，能口述"这个理论解决了什么问题？不用它会怎样？工作中如何落地？"

## 一句话口诀

> **SOLID** 是设计准则，**DDD** 是业务建模，**CAP** 是分布式选型，**CI/CD** 是工程实践，**测试金字塔**是质量保障。
