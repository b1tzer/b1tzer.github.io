# 企业内部问答系统 — 项目概览

---

## 项目简介

本项目是一个**公司内部知识问答平台**，类似于企业版 Stack Overflow，旨在沉淀团队知识、促进内部技术交流。用户可以在平台上提问、回答、搜索问题，并通过圈子（论坛）进行分类管理。

---

## 核心功能模块

```mermaid
mindmap
  root((企业问答系统))
    问答核心
      提问 / 回答
      点赞 / 点彩
      评论（叠楼回复）
      收藏
      Markdown 支持
    搜索系统
      全文搜索（ES）
      标签搜索
      圈子搜索
    圈子论坛
      创建 / 编辑圈子
      设置版主
      关联标签
      @用户
    用户中心
      个人问答 / 草稿
      活跃度统计
      删除自己的内容
    热门统计
      热门问题
      热门标签
      热门圈子
      活跃用户
    权限管理
      普通用户
      圈子版主
      系统管理员
```

---

## 技术栈

| 技术 | 用途 |
|------|------|
| **Spring Cloud（Eureka）** | 微服务注册中心，服务发现 |
| **Spring Cloud Gateway** | API 网关，统一路由与鉴权 |
| **MySQL** | 主要业务数据存储 |
| **Redis** | 缓存、计数器、分布式锁 |
| **Elasticsearch** | 全文搜索 |
| **Kafka** | 异步消息处理用户交互事件 |
| **Docker** | 四个后端服务容器化部署 |

---

## 微服务划分

```mermaid
flowchart TD
    Client["客户端"] --> Gateway["API 网关\nSpring Cloud Gateway"]
    Gateway --> UserService["用户服务\nuser-service"]
    Gateway --> QAService["问答服务\nqa-service"]
    Gateway --> ForumService["圈子论坛服务\nforum-service"]
    Gateway --> SearchService["搜索服务\nsearch-service"]

    UserService --> MySQL[(MySQL)]
    QAService --> MySQL
    ForumService --> MySQL
    SearchService --> ES[(Elasticsearch)]

    QAService --> Redis[(Redis)]
    UserService --> Redis

    QAService --> Kafka[[Kafka]]
    ForumService --> Kafka
    Kafka --> SearchService
    Kafka --> UserService
    
    Eureka["注册中心\nEureka"] -.->|服务注册发现| UserService
    Eureka -.->|服务注册发现| QAService
    Eureka -.->|服务注册发现| ForumService
    Eureka -.->|服务注册发现| SearchService
```
