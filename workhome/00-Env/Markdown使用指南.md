# Markdown 使用指南

## 1. 什么是 Markdown

Markdown 是一种**轻量级标记语言**，由 John Gruber 于 2004 年创建。它的核心理念是：**用纯文本写作，让内容本身比格式更重要**。

作为开发者，Markdown 几乎无处不在——GitHub README、技术博客、API 文档、Notion 笔记、飞书文档……掌握 Markdown 是每个开发者的基本功。

> 一句话：Markdown 让你用最少的格式标记，写出结构清晰、排版美观的文档。

---

## 2. 基础语法

### 2.1 标题

使用 `#` 号表示标题层级，`#` 越多层级越深（最多 6 级）：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

> 💡 **最佳实践**：一篇文档只用一个 `#` 一级标题，正文从 `##` 开始。标题层级不要跳跃（比如从 `##` 直接到 `####`）。

### 2.2 段落与换行

```markdown
这是第一段。段落之间用空行分隔。

这是第二段。
如果只是回车换行（没有空行），在标准 Markdown 中会被合并为同一段。

如果需要强制换行，可以在行尾加两个空格  
然后回车，这样就会换行但不分段。
```

### 2.3 文本样式

| 语法 | 效果 | 说明 |
|------|------|------|
| `**粗体**` | **粗体** | 强调重要内容 |
| `*斜体*` | *斜体* | 轻度强调 |
| `***粗斜体***` | ***粗斜体*** | 最强强调 |
| `~~删除线~~` | ~~删除线~~ | 表示废弃或修正 |
| `` `行内代码` `` | `行内代码` | 代码、命令、变量名 |
| `==高亮==` | ==高亮== | 部分渲染器支持 |

### 2.4 列表

**无序列表**（`-`、`*`、`+` 均可，推荐统一用 `-`）：

```markdown
- 第一项
- 第二项
  - 嵌套项（缩进 2 个空格）
  - 嵌套项
- 第三项
```

**有序列表**：

```markdown
1. 第一步
2. 第二步
3. 第三步
   1. 子步骤 a
   2. 子步骤 b
```

**任务列表**（GitHub / MkDocs 支持）：

```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 待办事项
```

效果：

- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 待办事项

### 2.5 链接与图片

```markdown
<!-- 行内链接 -->
[显示文本](https://example.com)

<!-- 带标题的链接（鼠标悬停显示） -->
[显示文本](https://example.com "链接标题")

<!-- 引用式链接（适合多次引用同一链接） -->
[显示文本][link-id]
[link-id]: https://example.com "链接标题"

<!-- 图片 -->
![替代文本](图片地址 "可选标题")

<!-- 图片 + 链接（点击图片跳转） -->
[![替代文本](图片地址)](链接地址)
```

### 2.6 引用

```markdown
> 这是一段引用。
> 可以多行。
>
> > 嵌套引用也是可以的。
```

效果：

> 这是一段引用。
> 可以多行。
>
> > 嵌套引用也是可以的。

### 2.7 分隔线

三种写法均可，推荐 `---`：

```markdown
---
***
___
```
